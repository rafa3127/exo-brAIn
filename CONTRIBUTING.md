# Contributing to exo-brAIn

exo-brAIn is an AI-assisted Obsidian vault template. Contributions are welcome — but note that **skill-specs in this repository are executed by AI agents with file system access**. This makes every contribution effectively executable code, and all changes receive security review accordingly.

## What we accept

- **New abstract skill-specs** — operations useful to any vault user, regardless of their domain
- **New area templates** — new area types beyond the current four (project, study, knowledge, personal)
- **Improvements to existing skills** — bug fixes, better procedures, new edge cases
- **Bug fixes** — corrections to documentation or procedures
- **Documentation improvements** — clearer explanations, better examples

## What we do NOT accept

- **Personal-specific skills** — skills that only work for a particular user's setup or domain (e.g., "generate my weekly kiosk report")
- **Edits to `_index.md` files** — these are auto-generated by skills, never manually edited
- **Edits to `_system/config.json`** — this is personal configuration (gitignored)
- **Content targeting a specific AI provider** — all skills must be portable across AI tools
- **Hardcoded personal values in templates** — use `{{placeholders}}` instead

## Conventions

### Frontmatter

All `.md` files require frontmatter. Required fields depend on file type:

| File type | Required fields | Notes |
|-----------|----------------|-------|
| Skill-spec | `skill`, `audience`, `version`, `last_updated` | `skill` must match filename |
| Template | `template_for`, `audience`, `doc_type` | `template_for` must match filename |
| Other docs | `audience`, `doc_type`, `version`, `last_updated` | — |

**Allowed values:**
- `audience`: `ai` (compressed, token-efficient) · `human` (readable) · `shared` (balanced)
- `doc_type`: `decision` · `log` · `checklist` · `architecture` · `instruction` · `index` · `note`
- `version`: positive integer, starting at 1

### Naming

- Files: **kebab-case** (lowercase, hyphens, no spaces). Example: `review-contribution.md`
- Temporal files: `YYYY-MM-DD-slug.md`
- Templates: named after their area type (e.g., `project.md`)

### Versioning

- New files start at `version: 1`
- When modifying an existing file, increment `version` and update `last_updated`

## How to contribute

1. **Fork** the repository
2. **Create a branch** from `main`:
   - `feat/description` — new features (skills, templates)
   - `fix/description` — bug fixes
   - `docs/description` — documentation only
   - `improve/description` — improvements to existing skills
3. **Make your changes** following the conventions above
4. **Self-review** — ask any AI agent to run the `review-contribution` skill (`_system/skill-specs/review-contribution.md`) on your changes
5. **Open a Pull Request** using the provided template

## Skill-spec structure

New skill-specs must follow the existing pattern:

```yaml
---
skill: your-skill-name
audience: ai
version: 1
last_updated: YYYY-MM-DD
depends_on: []
---
```

Required sections: `## Purpose`, `## When to use`, `## Procedure`.

See any existing skill in `_system/skill-specs/` for reference.

## Review process

1. An AI agent runs the `review-contribution` skill on the PR (maintainer-triggered or self-service)
2. Security scan checks for prompt injection, override attempts, and exfiltration patterns
3. Convention checks validate frontmatter, naming, dependencies, and content quality
4. Maintainer reviews the AI assessment and makes the final decision
5. At least one maintainer approval is required to merge

## Security

Skill-specs are markdown procedures that AI agents execute with file system access. This means:

- All contributions are scanned for **prompt injection** patterns (override attempts, system impersonation, hidden instructions, exfiltration)
- Skills that modify other skills, config, or system files are flagged for extra scrutiny
- When in doubt, the reviewer flags for human judgment rather than auto-approving

If you find a security vulnerability in an existing skill-spec, please open an issue with the `security-sensitive` label.
